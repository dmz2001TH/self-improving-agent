# 🚀 COLD_START.md — โหลดอันเดียว จำได้เลย

ไฟล์นี้คือ **snapshot ความจำ** ที่บีบแล้ว
Agent อ่านไฟล์เดียว = จำได้ทันที ไม่ต้องไล่อ่านทุกไฟล์

---

## วิธีใช้

### Option A: ใส่ใน system prompt (เร็วสุด)
```
คุณคือ Builder agent ที่ช่วย [อธิบายหน้าที่]

## Quick Context (โหลดแล้ว จำได้เลย)
[วางเนื้อหาด้านล่างตรงนี้]
```

### Option B: Agent อ่านเองตอนเริ่ม
```
ตอนเริ่ม session ใหม่ ให้:
1. อ่าน COLD_START.md ก่อน (ไฟล์เดียว)
2. ทำงานได้เลย
3. ถ้างานซับซ้อนค่อยอ่านไฟล์อื่นเพิ่ม
```

### Option C: Warm-up message (สั้นสุด)
```
ส่งข้อความนี้ให้ agent ตอนเริ่ม:
"wake up, load COLD_START.md, then tell me what you remember"
→ Agent จะโหลด context + ยืนยันว่าจำได้ ใช้เวลา ~2-3 วินาที
```

---

## ตัวอย่าง COLD_START.md

```markdown
# Builder Agent — Cold Start Memory

## ฉันคือใคร
- ชื่อ: Builder
- หน้าที่: ช่วยพัฒนาโปรเจกต์ [ชื่อโปรเจกต์]
- Stack: Python, [อื่นๆ]

## โปรเจกต์ปัจจุบัน
- Repo: [url]
- Branch: main
- Status: [กำลังทำอะไรอยู่]

## ความจำสำคัญ (ล่าสุด)
- [สิ่งที่ทำสำเร็จล่าสุด]
- [ปัญหาที่แก้ล่าสุด]
- [สิ่งที่ต้องทำต่อ]

## บทเรียน (จาก self-improving-agent)
- ห้าม [สิ่งที่เคยทำผิด]
- ต้อง [สิ่งที่ต้องเช็คก่อนทำงาน]
- ถ้าเจอ [error type] → แก้ด้วย [วิธี]

## ข้อจำกัด
- ห้าม [สิ่งที่ห้ามทำ]
- ถ้าไม่แน่ใจ → [วิธีขอความช่วยเหลือ]
```

## ทำอัตโนมัติด้วย Cron

ตั้ง cron job ให้ update COLD_START.md ทุกวัน:
```
ทุก end-of-day:
1. อ่าน memory/*.md ของวันนี้
2. สรุปเป็น bullet points สั้นๆ
3. เขียนทับ COLD_START.md ด้วยข้อมูลล่าสุด
4. วันหน้า agent เริ่มใหม่ → อ่านไฟล์เดียว → จำได้เลย
```

---

## เปรียบเทียบเวลา

| วิธี | เวลา boot |
|---|---|
| โหลด memory ทั้งหมดทุกไฟล์ | 5-10 วินาที |
| อ่าน COLD_START.md ไฟล์เดียว | 1-2 วินาที |
| ใส่ใน system prompt | 0 วินาที (จำได้เลย) |
| Session ไม่ตาย (tmux) | 0 วินาที (ไม่ต้อง boot) |
